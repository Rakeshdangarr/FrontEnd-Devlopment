Qus: What the deference between undefined and undeclare in JavaScript?
Ans:
undeclared undefined
  • In JavaScript, an undeclared variable is a variable that has not been declared or initialized using the var, let, or const keyword.
  • Undeclared variables are automatically global in scope, which means they can be accessed and changed anywhere.
  • In JavaScript, a variable is considered undefined if it has not been assigned a value.
  • This can happen if the variable has been declared but not initialized, or if it has been assigned the value undefined.
